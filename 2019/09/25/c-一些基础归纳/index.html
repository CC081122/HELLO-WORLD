c++一些基础归纳

Posted on 2019-09-25

引言

本文归纳了一些c++的基础语法

内容

  1. explicit:

    c++中，explicit只能用于构造函数中，不能发生隐式类型转换，只能显示的方式进行类型转换。

  2. 拷贝构造函数：

    形式：

    ClassExaple(const ClassExample &ce)，函数名称必须和类名称一致，必须的一个参数是是本类 型的引用变量。

    1. 调用拷贝构造函数的情况：

      • 对象以值传递的方式传入函数参数。（产生临时变量，函数执行完成后析构）
      • 对象以值的方式从函数返回。（return 的时候，编译优化时不会调用-O0）
      • 对象通过另一个对象进行初始化。

    2. 深拷贝与浅拷贝

      • 默认编译器会自动生成一个拷贝构造函数，这个函数很简单，使用“老的对象”的数据对“新对象”的成员一一赋值。
      • 默认拷贝构造函数不能处理静态数据成员。
      • 浅拷贝，对象复制的时候，简单的赋值。如果对象存在动态成员，浅拷贝只会复制指针，并不会生成内存空间。如下面代码，ex2中的p和ex1指向同一个地址空间。
* 深拷贝，对于上述p的内存空间，不能简单的赋值，需要重新分配空间，使得其指向的空间具有相同的内容。
* 防止默认拷贝发生，对象拷贝构造通常发生在按值传递时，可以通过声明一个私有拷贝构造函数。

  1. const：

    1. const int* p , p 为指向常量的指针，*p不能变
    2. int * const p, p 为一个常量指针，p的指向不能变。
    3. const int 和 int const 等价，常量int。
    4. const int& 修饰引用时可以用任意值进行初始化。
    5. const 修饰函数， const fun(const arg) const, 第一个为返回值不能修改，第二个为参数 为常量，第三个为表示调用常量函数，不能修改成员变量，常量对象只能调用常量函数。
    6. const 与 #deine区别，const 便于进行类型检查，const避免不必要的内存分配

  3. new 操作符

    1. new operator就是new操作符，A *a = new A 执行如下3个过程
      • 调用operator new 分配内存， operator new (sizeof(A))
      • 调用函数生成类对象的构造函数
      • 返回相应指针
    2. operator new时函数，可以被重载。

  4. 函数重载

    1. 函数重载必须在下列一个或两个方面有所区别
      • 函数参数个数不同
      • 函数的参数类型不同或者参数类型顺序不同

  5. class 和 struct的大小

    1. 64位机器上, int 4 个字节， void* 8个字节。没有成员变量的class和struct具有1个大小，为了区别使实例出来的对象的地址不同（是否真的有实际意义？）。成员变量char data[0]和char data[]不占空间, char *data占用一个指针的大小。

  1. 变长指针数组

    1. 使用char data[0]或者char data[1]可以构建变长指针数组。

      结构中的最后一个元素允许是未知大小的数组，这就叫做柔性数组成员，但结构中的柔性数组成员前 面必须至少一个其他成员。柔性数组成员允许结构中包含一个大小可变的数组。sizeof返回的这种结构 大小不包括柔性数组的内存。包含柔性数组成员的结构用malloc ()函数进行内存的动态分配，并且分 配的内存应该大于结构的大小，以适应柔性数组的预期大小

  2. SIGABRT的可能原因

    1. double free
    2. 堆越界
    3. assert

  3. C++四种强制类型转换

    1. const_cast
      • const 指针转换为非 const 指针，指向对象不变
      • const 引用被转换为非 const 引用，指向对象不变
      • 多用于修改指针类型
    2. static_cast
      • 基本与 c 语言的指针转换相同，相关类型的转换，编译器会进行截断，补齐等操作。
      • 派生类转基类安全，基类转派生类不安全
      • 不能去去掉 const, volatile
      • c++ 任意的隐式类型转换都是static_cast完成的。
    3. dynamic_cast
      • 用户基类和派生类的转换，派生类可转基类。(上行转换)
      • 基类转派生类时，会检查是否被转换的指针是否指向派生类，否则抛出错误或者返回nullptr。 （下行转换）
    4. reinterpret_cast
      • 强制类型转换符用来处理无关类型的转换。
      • 多用于将一个类型转换为另一个类型。
      • 只是执行了比特位拷贝，做出另一种解释。

  4. 纯虚函数

    1. 纯虚函数是一种特殊的虚函数，在许多情况下，在基类中不能对虚函数给出有意义的实现，而把它声明为纯虚函数。
    2. 包含存储函数的类称为抽象类，其不能实例化。

  5. Friend Classes(友元类）

    • Firend Classes 申明后，该class 可以直接访问该类的private成员变量。
